# Working style

How to propose approaches and make decisions on this project.

- Prefer the simplest mechanism that meets the need; don't over-engineer — I'll
  ask for more rigor when I want it.
- When proposing an approach, give alternatives and a recommendation, and explain
  the trade-offs in plain terms.
- Capture decisions durably (in `docs/`) as they're made, not just in
  conversation.
